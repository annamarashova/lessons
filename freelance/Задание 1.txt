Заказчик просит сделать скрипт, который будет читать сообщения с сигналами из исходного телеграм-канала
(source_channel) и перенаправлять их в destination channel в новом формате.

Основная сложность состоит в том, чтобы сделать преобразование сообщения из формата source channel в формат
destination channel.

В качестве ТЗ заказчик прислал примеры сообщений из source channel, и для каждого примера написал, каким должно
быть сообщение в destination channel.

Твоя задача -- глядя на примеры сообщений от заказчика, реализовать функцию "source_to_destination_text".
На вход данной функции будет подаваться текст сообщения из source channel.
Функция будет извлекать из текста необходимые данные и формировать сообщение для destination channel.

Требования к функции source_to_destination_text:
- принимает на вход текст из исходного канала (входной параметр source_text)
- извлекает из исходного текста данные, которые понадобятся для формирования сообщения в destination channel
- формирует и возвращает сообщение для destination channel

Требования к выполнению задания:
- функцию нужно реализовать в файле solution_1.py
- данные от заказчика в сыром виде лежат в файле client_raw_data.pdf
- примеры текстов сообщений от заказчика оформлены в виде тест-кейсов (в файле client_cases.py)
- для твоей будущей функции написаны юнит-тесты по всем 4-м тест-кейсам (в файле test_solution_1.py)
- минимальный критерий выполнения задачи -- проходят все 4 теста из файла test_solution_1.py
(запускать с помощью pytest также, как в ДЗ епам)
- при разработке тестировать функцию можно прямо в файле solution_1.py (так же, как делали в ДЗ епам),
а также запускать тест-кейсы -- на твое усмотрение.

И самое главное: в файле "Подсказки.md" есть ответы на многие вопросы, которые могут возникнуть, в частности:
- как установить зависимости из requirements.txt
- как извлечь данные из исходного текста
- как можно сформировать сообщение для destination channel

Используй файл на своё усмотрение, и успехов! Ты справишься!
